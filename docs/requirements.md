# 要件定義書 — 補助金・助成金マッチングサイト

## 1. 機能要件

### FR-01: 一問一答マッチングフロー（コア機能）

| ID | 要件 | 優先度 |
|---|---|---|
| FR-01-01 | 1画面1問の形式で質問を表示する | 必須 |
| FR-01-02 | 進捗バーを常時表示し、完了率をリアルタイム更新する | 必須 |
| FR-01-03 | 「戻る」ボタンで前の質問に戻れる | 必須 |
| FR-01-04 | 回答データをブラウザ側（JavaScript）で保持する | 必須 |
| FR-01-05 | 最終問でメールアドレスを取得する（リード獲得） | 必須 |
| FR-01-06 | 回答完了後、マッチング結果画面に遷移する | 必須 |

**質問項目:**

| # | 質問 | 入力形式 |
|---|---|---|
| Q1 | 会社の所在地 | 都道府県プルダウン |
| Q2 | 業種 | プルダウン（大分類 → 中分類） |
| Q3 | 従業員数 | ラジオボタン（1-5名/6-20名/21-50名/51-100名/101名以上） |
| Q4 | 資本金 | ラジオボタン（300万円未満/300万-1,000万円/1,000万-3,000万円/3,000万-1億円/1億円以上） |
| Q5 | 現在の課題（複数選択可） | チェックボックス（設備投資/IT化・DX/人材採用/海外展開/研究開発/事業承継） |
| Q6 | 年間売上規模 | ラジオボタン（1,000万円未満/1,000万-5,000万円/5,000万-1億円/1億-5億円/5億円以上） |
| Q7 | 補助金申請経験 | ラジオボタン（はい/いいえ） |
| Q8 | メールアドレス | テキスト入力（バリデーション付き） |

### FR-02: マッチング結果表示

| ID | 要件 | 優先度 |
|---|---|---|
| FR-02-01 | 回答内容に基づき該当補助金をリスト表示する | 必須 |
| FR-02-02 | 各補助金に適合度バッジ（高/中/低）を表示する | 必須 |
| FR-02-03 | 補助金名、最大金額、概要、申請期限を表示する | 必須 |
| FR-02-04 | 「詳しく見る」リンクで外部サイト（公式）に遷移する | 必須 |
| FR-02-05 | 結果画面下部に相談フォームへの導線を設置する | 必須 |

### FR-03: 補助金データ管理（カスタム投稿タイプ）

| ID | 要件 | 優先度 |
|---|---|---|
| FR-03-01 | カスタム投稿タイプ「補助金」を登録する | 必須 |
| FR-03-02 | カスタムフィールド: タイトル、金額（最大）、補助率、対象業種、対象地域、対象従業員規模、対象資本金、対象課題、申請期限、公式URL、概要 | 必須 |
| FR-03-03 | WordPress 管理画面から CRUD 操作できる | 必須 |
| FR-03-04 | CSV 一括インポート機能 | 任意 |

### FR-04: リード管理

| ID | 要件 | 優先度 |
|---|---|---|
| FR-04-01 | 回答データ（メールアドレス + 全回答）をDBに保存する | 必須 |
| FR-04-02 | 管理画面からリード一覧を閲覧できる | 必須 |
| FR-04-03 | リードごとにマッチング結果を確認できる | 任意 |
| FR-04-04 | CSV エクスポート機能 | 任意 |

### FR-05: お問い合わせフォーム

| ID | 要件 | 優先度 |
|---|---|---|
| FR-05-01 | 会社名、担当者名、メールアドレス、電話番号、相談内容を入力できる | 必須 |
| FR-05-02 | 送信時に管理者へメール通知する | 必須 |
| FR-05-03 | 送信完了画面を表示する | 必須 |

### FR-06: トップページ

| ID | 要件 | 優先度 |
|---|---|---|
| FR-06-01 | ヒーローセクション（キャッチコピー + マッチング開始ボタン） | 必須 |
| FR-06-02 | サービス概要セクション（3ステップ説明） | 必須 |
| FR-06-03 | 新着補助金情報セクション | 任意 |
| FR-06-04 | よくある質問セクション | 任意 |

### FR-07: WordPress REST API

| ID | 要件 | 優先度 |
|---|---|---|
| FR-07-01 | 補助金データ取得エンドポイント（GET） | 必須 |
| FR-07-02 | マッチング実行エンドポイント（POST: 回答データ → マッチ結果） | 必須 |
| FR-07-03 | リード保存エンドポイント（POST） | 必須 |

## 2. 非機能要件

### NFR-01: パフォーマンス

| ID | 要件 | 目標値 |
|---|---|---|
| NFR-01-01 | ページ初期表示速度 | 3秒以内 |
| NFR-01-02 | マッチング結果表示速度 | 2秒以内 |
| NFR-01-03 | 一問一答の画面遷移 | 即時（クライアントサイド） |

### NFR-02: セキュリティ

| ID | 要件 |
|---|---|
| NFR-02-01 | SSL/TLS（HTTPS）必須 |
| NFR-02-02 | WordPress 本体・プラグインの定期アップデート |
| NFR-02-03 | REST API のレート制限（brute force 対策） |
| NFR-02-04 | メールアドレス等の個人情報は適切に保護する |
| NFR-02-05 | CSRF / XSS 対策（WordPress nonce 活用） |

### NFR-03: 可用性

| ID | 要件 | 目標値 |
|---|---|---|
| NFR-03-01 | サービス稼働率 | 99.5% 以上 |
| NFR-03-02 | バックアップ | 日次（DB + ファイル） |

### NFR-04: 対応環境

| ID | 要件 |
|---|---|
| NFR-04-01 | レスポンシブデザイン（モバイルファースト） |
| NFR-04-02 | Chrome / Safari / Edge / Firefox 最新版 |
| NFR-04-03 | iOS Safari / Android Chrome |

### NFR-05: SEO / マーケティング

| ID | 要件 |
|---|---|
| NFR-05-01 | OGP メタタグ設定 |
| NFR-05-02 | Google Analytics 連携 |
| NFR-05-03 | Google Tag Manager 連携 |
| NFR-05-04 | 構造化データ（JSON-LD） |

### NFR-06: 運用

| ID | 要件 |
|---|---|
| NFR-06-01 | Docker Compose でローカル環境構築可能 |
| NFR-06-02 | WordPress 管理画面で非エンジニアが補助金データを更新可能 |
